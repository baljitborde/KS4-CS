Question
Read the code, then select what will happen

A student writes the following Python program:

def calculate_grade(score): if score >= 70: grade = "A" elif score >= 55: grade = "B" else: grade = "C" return grade student_score = input("Enter score: ") result = calculate_grade(student_score) print("Your grade is: " + result)

The student runs the program and types 80 when prompted. What will happen, and why?

✗ Not quite. This would be the correct output if the program ran without errors — but it will not get that far. Think about what data type input() always returns. Can you use >= to compare a string and an integer?
✗ Not quite. The else branch does not always run — it only runs when the conditions above are False. But there is a more fundamental problem. What does input() return? What data type is student_score when it reaches calculate_grade()?
✓ Correct! input() always returns a string, so student_score holds the string "80", not the integer 80. When calculate_grade() tries to evaluate "80" >= 70, Python cannot compare a string with an integer and raises a TypeError.

The fix is a single change on the input line:
# Change this: student_score = input("Enter score: ") # To this: student_score = int(input("Enter score: "))

✗ Not quite. Python does not compare strings and integers using >= at all — it raises a TypeError immediately. String comparison rules only apply when both values are strings.
